# Быстрая настройка удаленного доступа

## Самый простой способ: DuckDNS (5 минут)

### Шаг 1: Регистрация в DuckDNS
1. Откройте https://www.duckdns.org
2. Войдите через Google/GitHub/Twitter
3. Создайте поддомен (например: `mycolumn`)
4. Получите домен: `mycolumn.duckdns.org`
5. Скопируйте токен

### Шаг 2: Настройка на роутере
1. Войдите в роутер (обычно http://192.168.1.1)
2. Найдите "DDNS" или "Динамический DNS"
3. Выберите "DuckDNS"
4. Введите:
   - Домен: `mycolumn.duckdns.org`
   - Токен: (ваш токен из шага 1)
5. Сохраните

### Шаг 3: Проброс порта
1. В роутере найдите "Port Forwarding"
2. Добавьте:
   - Внешний порт: `80`
   - Внутренний IP: `192.168.1.XXX` (IP вашего ESP32)
   - Внутренний порт: `80`
   - Протокол: `TCP`
3. Сохраните

### Шаг 4: Настройка в приложении
1. Откройте приложение
2. Настройки устройства → Добавить устройство
3. Введите:
   - **Хост:** `mycolumn.duckdns.org`
   - **Порт:** `80`
   - **HTTPS:** Выключено (если не настроен SSL)
   - **Логин/Пароль:** (если включена авторизация на ESP32)
4. Сохраните

### Готово! ✅

Теперь вы можете подключаться к устройству из любой точки мира.

## Альтернатива: Узнать IP и использовать напрямую

### Шаг 1: Узнать публичный IP
1. Откройте https://whatismyipaddress.com
2. Скопируйте IPv4 адрес (например: `123.45.67.89`)

### Шаг 2: Проброс порта
(См. Шаг 3 выше)

### Шаг 3: Настройка в приложении
1. Введите:
   - **Хост:** `123.45.67.89` (ваш публичный IP)
   - **Порт:** `80`
   - **HTTPS:** Выключено

**⚠️ Проблема:** IP может измениться при переподключении интернета. Используйте DDNS для постоянного доступа.

## Проверка работы

После настройки проверьте:

1. **Из браузера:**
   ```
   http://mycolumn.duckdns.org
   ```
   Должна открыться веб-страница ESP32

2. **Из приложения:**
   - Добавьте устройство
   - Нажмите "Проверить подключение"
   - Должно показать "Подключение успешно"

---

## Решение проблем

### Не подключается из интернета
- ✅ Проверьте, что порт проброшен на роутере
- ✅ Проверьте, что ESP32 включен и подключен к WiFi
- ✅ Проверьте, что в локальной сети доступ работает
- ✅ Проверьте файрвол на роутере

### IP адрес изменился
- ✅ Используйте DDNS вместо прямого IP
- ✅ Проверьте, что DDNS обновляется (в панели DuckDNS)

### Ошибка подключения
- ✅ Проверьте правильность домена/IP
- ✅ Проверьте порт
- ✅ Проверьте логин/пароль (если включена авторизация)
